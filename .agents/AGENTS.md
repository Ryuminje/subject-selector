# 고교학점제 선택과목 수요조사 및 분반 배정 자동화 시스템 (Subject Selector)

## 📌 프로젝트 소개
이 프로젝트는 **고교학점제 선택과목 수요조사 데이터를 기반으로 시수 산출, 시간표 작성, 분반 배정까지의 과정을 자동화**하는 웹 기반 시스템입니다.
교사가 엑셀 데이터를 업로드하면 즉각적으로 교육과정 편제와 시간표를 작성하고, 학번순 분반 등 번거로운 수작업을 시스템이 자동으로 처리합니다.
이 문서는 추후 이 프로젝트를 유지보수하거나 새 기능을 추가할 **AI 에이전트(Agent)**를 위한 가이드(Rules/Instructions) 역할을 합니다.

## 🛠 주요 기능 및 워크플로우

### [수요조사 기반 시수 산출 탭]
1. **교육과정 편성표 업로드**
   - 엑셀 파일을 업로드하여 2학년/3학년별 전체 과목 편제를 파싱합니다.
   - **동적 열 탐색:** 엑셀 양식이 다르더라도 '구분', '교과(군)', '과목', '운영학점'을 동적으로 스캔합니다.
   - **특수 기호 분리:** 과목명에 `↔` 기호가 있는 경우(예: 물리학Ⅰ↔화학Ⅰ) 두 과목으로 분리합니다.
   - **학기별 운영학점 산출:** '운영학점' 열을 무시하고, 실제 학기(1학년 1학기 등) 아래에 적힌 숫자를 가져와 학점으로 간주합니다.
2. **수요조사 파일 업로드**
   - 학생들이 제출한 수강 희망 과목 엑셀 데이터를 파싱합니다.
3. **분석 및 위계성 검사**
   - 학생별 선택 학점 수(시수)가 조건에 맞는지 확인합니다.
   - **원천 데이터 기반 교과군 판별:** 하드코딩된 규칙 없이, 오직 1단계에서 업로드한 교육과정 엑셀 원본의 교과군 기준만을 참고해 기초/사회/과학/기타 학점을 정확히 집계합니다.

### [선택과목 변경 탭]
1. **1단계: 학생 선택 데이터 업로드 (타임별 데이터)**
   - 열 헤더가 과목명이고 각 셀이 A, B, C 등 선택 그룹(타임)을 나타내는 **시간표 편성용 수요조사 엑셀(sample3.xlsx 형태)**을 파싱합니다.
2. **2단계: 타임별 시간표 입력**
   - 2학년/3학년별로 A, B, C 등 타임별 시간표(행)와 각 반(열)을 관리하는 동적 Grid(격자)를 제공합니다.
   - 격자의 각 칸은 위(과목명), 아래(교사명)로 나뉘어 있으며, **엑셀 복사/붙여넣기를 완벽하게 지원**합니다.
3. **3단계: 타임별 선택과목 명단 산출 (변경 전 자동 분반 배정)**
   - 1단계의 학생 데이터와 2단계의 시간표를 결합하여 **과목별/반별 학생 명단**을 산출합니다.
   - **강력한 자동 분반 기능:** 만약 시간표 Grid에 동일한 과목이 여러 반으로 나뉘어 기재될 경우(예: 심리학1, 심리학2, 심리학 3반), 정규식을 이용해 원본 과목명(심리학)을 자동 추출합니다.
   - 추출된 과목을 선택한 전체 학생을 **학번순(오름차순)으로 정렬**한 뒤, 반 개수에 맞춰 **정확히 등분하여 배정**합니다. (예: 25명, 2개 반 -> 1반 13명, 2반 12명)
4. **4단계: 선택과목 변경 신청 내역 및 자동 검증**
   - 학생별로 **과목 변경 내역(변경 전 과목 -> 변경 후 과목)**을 입력받습니다. 여러 과목을 변경하는 학생을 위해 `+` 버튼을 통해 묶음 입력을 지원합니다.
   - **자동 교체 로직(2단계 우회 교환 지원):** 변경 후 과목이 변경 전 과목과 동일한 타임에 있다면 1단계 교체를 즉시 승인합니다. 만약 다른 타임에 있다면, 해당 과목이 개설된 모든 타임을 스캔하여 다른 빈 타임으로 밀어내는 형태의 **2단계 우회 교체(Swap)**가 가능한지 검증합니다.
   - 불가피하게 교환이 불가능한 경우 빨간색 `(불가)` 태그와 함께 불가능한 사유(예: "B타임에 개설되지 않음")를 툴팁으로 안내하여 직관적인 피드백을 제공합니다.
   - 로마 숫자(Ⅰ, Ⅱ, Ⅲ)와 영문 알파벳(I, II, III), 띄어쓰기를 자동으로 정규화하여 비교하는 **Fuzzy Matching** 기능이 내장되어 있습니다.
5. **5단계: 변경 후 타임별 선택과목 명단 산출**
   - 4단계에서 검증 완료(Success)된 변경 내역만을 바탕으로 학생 명단을 재배정하여 최종 결과를 보여줍니다.
   - 변경된 학생의 이름은 초록색으로 강조되며, 마우스를 올리면 "어떤 과목에서 무엇으로 바뀌었는지"를 상세한 툴팁으로 확인할 수 있습니다.

## 💻 기술 스택 및 구현 세부사항
- **Framework:** Next.js (TypeScript, React)
- **Styling:** Tailwind CSS (다크 모드 베이스, Glassmorphism, 모던 프리미엄 UI 지향)
- **Icons:** lucide-react
- **Data Parsing:** xlsx-js-style (또는 xlsx)을 사용한 바이너리 엑셀 파싱 로직

## ⚠️ AI 에이전트를 위한 향후 개발 가이드라인 (Rules)
- **상태 관리:** 2학년과 3학년 데이터(changeActiveGrade)는 각각 분리된 상태로 유지보수해야 합니다.
- **분반 정규식 로직:** page.tsx 내의 명단 산출 로직 중 `cleanBase`와 `cleanChosen`을 통해 모든 공백과 기호를 무시하고 매칭하는 로직을 함부로 축소하지 마세요. 이는 다양한 사용자 입력값(타이포그래피)을 보정하기 위한 필수 로직입니다.
- **2단계 교환 검증 로직:** 4단계의 `adjustmentLog`는 반드시 하나의 과목만 찾는 것이 아니라 `findSlotsWithSubject`를 통해 과목이 개설된 **모든 타임을 전부 탐색**하여 가능한 Swap 경우의 수를 모두 시도해야 합니다.
- **연쇄적 상태 업데이트:** 한 학생이 여러 과목을 동시에 변경할 수 있으므로, 4단계 교환 로직에서 `studentSchedules`와 같은 '임시 시간표(Working Copy)'를 만들어 변경이 발생할 때마다 실시간으로 시간표 상태를 갱신해야 충돌 에러가 발생하지 않습니다.
- **원본 데이터 보존:** 교육과정 편제 파싱 시 로마자(Ⅰ, Ⅱ)를 아라비아 숫자로 강제 변환하지 않고 원본 그대로 표기합니다.
- **UI/UX 기준:** 기능 추가 시 Tailwind CSS를 이용해 **직관적이고 미려한 UI(hover 애니메이션, 트랜지션, 색상 조화)**를 필수로 유지하세요. 단순한 형태의 테이블이나 버튼은 지양합니다.

## 🚀 배포(Deployment) 가이드라인 (NAS / Docker 환경)
- **메모리 최적화 (OOM 방지):** Synology NAS 등 저사양 기기에서 배포할 때 `npm ci`나 `npm install` 과정에서 "Exit handler never called!" 메모리 초과 에러가 발생할 수 있습니다. 이를 방지하기 위해 `Dockerfile`은 무조건 `node:20-slim` 기반 이미지를 사용하고 패키지 설치는 가벼운 `yarn install`을 사용합니다. (`node-alpine`의 musl libc 충돌 버그 우회)
- **네트워크 설정 (DNS 해상도 오류 방지):** NAS 도커 환경에서 라이브러리 다운로드 시 `getaddrinfo EAI_AGAIN` 인터넷 연결 오류가 발생하는 경우가 많습니다. 이를 해결하기 위해 `docker-compose.yml`의 `build` 섹션에는 반드시 `network: host`를 포함해야 합니다.
- **배포 자동화 스크립트:** 프로젝트 최상단에 있는 `deploy.sh` 스크립트를 사용하여 로컬에서 NAS로 파일을 전송(rsync/scp)하고, SSH로 원격 접속하여 `sudo docker compose up -d --build`를 실행하는 구조로 되어있습니다. (사용자 비밀번호 입력 필요)
- **Next.js Standalone 빌드:** Next.js 최적화 빌드를 위해 `next.config.ts`에 `output: 'standalone'` 설정이 켜져 있습니다.
